点餐系统点菜界面数据需求

1.基础数据

菜品数据：

  • ID
  • name (菜名)
  • price (价格)
  • src (图片)
  • introduction (简介)
  • ingredients (材料清单)
  • comment (用户评价)
  • star (点赞数)

  • sales (成交数据)

  • discount (促销折扣)

  • class（分类）

用户数据：

  • tableID (桌号)

后厨数据：

  • status (订单状态) 【等待烹饪，烹饪中，正在配送】

    与菜单点菜功能无关的数据先不使用。

2.各部分功能需求

2.1 点菜加入购物车

需求数据：

  • ID
  • name
  • price
  • count（当前数量）
  • Sumprice（总价）
  • discount

功能描述：

  • 点击添加按钮后在购物车显示【name】【price】【count+1】 并更新【Sumprice】

  • 点击删除按钮后【count-1】,更新【总价】

    只有当（count > 0 ）时删除操作才可执行

    (count == 0) 清除购物车内该菜品条目

    (count == 0) 时点击删除按钮，弹出报错提示

  • 加入菜单后，各菜品的数量可以自由调整，即更新【count】

  • 【Sumprice】计算时需要考虑折扣，Sumprice = price * discount

该功能提供【ID，name】，希望获得【price, count,Sumprice】

2.2 菜单展示–推荐位

需求数据：

  • star

功能描述：

  • 每次展示菜单的时候，自动将star最高的两项展示在菜单顶部
  • 顾客可以点击按钮【star+1】，再次点击【star-1】，单个用户只能点赞一次，最终菜单界面显示的是点赞总数。

该功能需要收集各菜的star数据，并且根据各菜品的star排序。实时获取前两位展示。

2.3 菜单展示–侧边栏

需求数据：

  • class
  • star
  • sales
  • discout

功能描述：

  • 根据【class】对菜品分类，通过侧边栏tag的切换展示不同种类的菜。
  • 侧边栏除了class分类之外，加上【推荐】【促销】
  • 【推荐】:分别取【star】【sales】前三进行展示
  • 【促销】:将【discount】不是1的的菜放入此类，并加上促销标记。

该功能需要在用户操作时候，收集需求的数据，展示需求同 2.2

2.4 菜品详情页

需求数据：

  • ID
  • src
  • price
  • name
  • introduction
  • ingredients
  • comment
  • sales
  • star
  • discount

功能描述：

  • 在用户点击菜单上单个菜品图片时进入详情页
  • 详情页向用户展示更为具体的信息，用户点击时通过调取点击时card的数据，自动生成详情页。
  • 详情页也具备点菜的功能，即【2.1的所有功能。】
  • 在详情页，用户可以进行评论，评论需存入数据库

该功能在详情页每次创建时需要调用所有上述需求数据，并且在之后对用户的【star】，【comment】进行存储。

2.5 支付

需求数据：

  • tableID
  • Sumpice
  • price
  • name
  • count
  • sales

功能描述：

  • 在用户点击结账按钮后，弹出账单一览，包括详细条目和总价。此处数据可以用购物车的接口，除了展示形式，内容基本一致。
  • 在用户确认支付成功之后，菜单内所有菜的【sales += count】

该功能使用时，基本可以复用购物车数据，主要操作为【sales】的更新

2.6 用户菜单界面

需求数据：

  • ID
  • name
  • price
  • count（当前数量）
  • Sumprice（总价）
  • discount
  • status

功能描述：

  • 用户可在此处查看自己的完整菜单
  • 点击单个菜品，可以查看菜品【status】
  • 可以对菜单进行调整，如要加菜，数据需求与2.1一致
  • 要删除某个菜，需要根据【status】，若已经在烹饪或者配送途中无法进行此操作

需要获取status的值，展示所点菜的当前进度。
